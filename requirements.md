# 📋 요구사항 정의서 (Requirements)

## 기능 요구사항

### FR-01: 일기 작성
- 사용자는 자유 텍스트로 일기를 작성할 수 있다
- 최소 10자, 최대 2000자 제한
- 글자 수 실시간 표시
- 유효하지 않은 입력 시 제출 버튼 비활성화

### FR-02: AI 감정 분석
- 일기 제출 시 Amazon Bedrock Claude 모델을 호출하여 분석
- 하루 요약 (2~3문장, 담백한 톤)
- 위로/응원 메시지 (2~3문장, 반말, 공감 우선)
- 음악 추천 파라미터 (energy, valence, vibe, color, searchQueries)
- AI 응답의 마크다운 코드블록 래퍼 자동 제거 후 JSON 파싱

### FR-03: 음악 플레이리스트 추천
- AI가 생성한 searchQueries(3개)로 YouTube Data API 검색
- 각 키워드당 최대 4곡, 총 10~12곡 추천
- 중복 영상 ID 자동 제거
- YouTube 검색 실패 시에도 나머지 결과(요약, 위로 메시지)는 정상 표시

### FR-04: 음악 재생
- 플레이리스트에서 곡 클릭 시 YouTube iframe으로 재생
- 하단 MiniPlayer에서 현재 재생 곡 정보 표시
- 재생/일시정지 토글 기능 (MiniPlayer + TrackList 모두)
- 현재 재생 중인 곡을 다시 클릭하면 일시정지

### FR-05: 감성 비주얼
- 감정 color 값에 따라 배경 그라데이션 변경 (5가지 색상)
- energy 값에 따라 파티클 애니메이션 속도/밀도 조절
- tsparticles의 initParticlesEngine을 통한 엔진 초기화

### FR-06: 기록 관리
- localStorage에 모든 일기 엔트리 저장
- 기록 리스트: 최근순 카드 형태 표시
- 캘린더 뷰: 월별 달력에 일기 작성일 감정 색상 점 표시
- 날짜 클릭 시 해당 기록 상세 페이지로 이동
- 기록 상세: 원본 일기, 요약, 위로 메시지, 플레이리스트 재확인

### FR-07: 시드 데이터
- 앱 최초 로드 시 localStorage에 데이터가 없으면 샘플 일기 5개 자동 주입
- 다양한 감정(행복, 우울, 분노, 평온, 그리움)을 포함

## 비기능 요구사항

### NFR-01: 성능
- AI 응답 3초 이내
- 페이지 초기 로드 1초 이내
- Vite 빌드 최적화 (코드 스플리팅)

### NFR-02: 반응형 디자인
- 모바일 우선 설계
- 데스크톱 max-width 제한 (max-w-xl, max-w-2xl)

### NFR-03: 접근성
- 모든 버튼에 aria-label 제공
- 키보드 네비게이션 지원
- 스크린리더 호환 (role, aria 속성)

### NFR-04: 보안
- API 키는 서버사이드에서만 사용 (환경변수)
- .env 파일은 .gitignore로 제외
- 클라이언트에 시크릿 노출 없음

### NFR-05: 에러 처리
- Bedrock API 실패 시 사용자 친화적 에러 메시지
- YouTube 검색 실패 시 graceful degradation (나머지 결과는 표시)
- 네트워크 에러 시 재시도 안내

### NFR-06: 브라우저 호환
- Chrome, Safari, Firefox 최신 2버전 지원
